# YOLO11x-pose 模型保留说明与未来用途规划

> 写于 2026-07-09。当时的背景：Tier 2 高精度检测路径的代码已从系统中移除，但 `yolo11x-pose.pt` 模型文件（31MB）特意保留在 GPU 服务机 `/home/ubuntu/zhiguo/person_id/models/` 下（legacy_backup 里有硬链接副本），`models/readme.txt` 中已加保留注记。本文记录"为什么留着它"以及"将来什么时候、怎么用回来"。

## 背景：现状是什么

系统目前全程只用 **yolo11n-pose**（轻量版）做人体检测。它是整条识别流水线的最前端，每帧视频先过它，一次推理同时产出**人体框**和 **17 个 COCO 关键点**（就是鼻、眼、耳、肩、肘、腕、髋、膝、踝这 17 个骨骼点的位置）。

这些输出喂给了很多下游环节：

- 人体框 → 追踪引擎（维持 track_id）、裁出人体图送人脸识别（AdaFace）和人体重识别（SOLIDER ReID）；
- 关键点 → 姿态朝向分类（正面/左侧/右侧/背面四个桶，底库特征按桶存储）、判断是否露脸（决定走不走人脸链路）、入库质量评分、身体比例计算（肩髋比、躯干腿比等）。

**yolo11x-pose 是同系列的重型版**：模型更大、推理更慢，但检测框和关键点都明显更准，尤其在小目标、遮挡、侧身/背身这些困难场景下。按 `src/tier1/detection/yolo_pose.py` 开头注释里的原始设计，它本来是 Tier 2 身份确认阶段的高精度检测器（n 版负责逐帧实时追踪，x 版只在关键帧上出更准的结果），这条代码路径后来被删掉了，只剩模型文件。

## n 版的弱点在哪（为什么可能需要 x 版）

轻量模型换来的速度，代价是精度。n 版在以下场景关键点噪声大、容易丢检：

- **侧身、背身**：关键点置信度低、位置漂移，姿态桶可能分错；
- **远距离小目标**：测试设备实际推 720p，人离摄像头远时像素高度小，n 版漏检率上升；
- **遮挡**：家庭场景常见——人被沙发、餐桌半挡，或者蹲下、躺卧等非站立姿态；
- **儿童**：目标小，和"远距离"问题叠加。

而系统里恰好有一批**依赖关键点精度**的计算：姿态桶分类、身体比例、入库质量分。这些都在"入库"环节汇聚——底库存什么样的特征，直接决定识别准确率的天花板。

## 未来可能的四个用途（按价值排序）

### 1. 入库时刻的"精修"（最值得做）

入库是低频事件：每个人每个姿态桶只挑质量最高的几帧存进底库（见 `orchestrator._update_gallery`）。可以在这一刻用 x 版对候选帧**重新检测一遍**，用更准的关键点重算：

- 姿态桶分类 → 不分错桶（分错桶的特征会污染底库，匹配时拿正脸特征去比侧脸）;
- 身体比例 → 帧间更稳定，作为身份特征更可信；
- 入库质量分 → 更可信的择优。

代价是每次入库多几十毫秒的一次性推理，不碰逐帧热路径，多家庭并发下也扛得住。这实际上就是恢复原始的 Tier 2 设计，只是范围收窄到入库时刻。

### 2. 让"身体比例"真正成为可靠的辅助识别特征

家庭场景大量时间看不到脸（背影、侧身、低头），这时人体 ReID + 身体比例是识别主力。但 n 版算出的肩髋比、躯干腿比帧间抖动大，区分度打折扣。如果将来想用身体比例做更强的判别（例如区分身高相近的两个大人、或大人 vs 小孩），大概率需要 x 版关键点的精度。此用途是用途 1 的自然延伸。

### 3. 自动标注与评测基准（配合训练机，建议最先做的验证）

训练机（8.145.38.125）可以发挥作用：

- **评测**：拿 x 版当"接近真值"的基准，在自己的真实录像上量化 n 版的漏检率和关键点误差。先测量，才知道 n 版的差距是否真的构成瓶颈、值不值得做用途 1/2；
- **自动标注**：如果差距明显，可以用 x 版批量给家庭室内场景数据打伪标签（伪标签 = 用强模型的预测结果当训练标签），微调一个更适合本场景（720p、儿童、家具遮挡）的轻量模型，替换通用的 n 版。

这一步成本最低、不改线上代码，且它的结论决定其他用途做不做。

### 4. 疑难帧兜底（谨慎，优先级最后）

n 版丢检但轨迹预期人还在（蹲下、被家具半挡）时，用 x 版在 ROI（预期区域的裁剪图，比全帧小得多）上补检一次。可行，但发生在热路径上，会引入尾延迟，多家庭并发下必须限流。除非丢跟踪问题被实测证明严重，否则不建议先做这个。

### 不现实的用法：全帧替换 n 版

多路摄像头逐帧跑 x 版，推理开销大约翻好几倍，GPU 服务机上还跑着其他一堆服务，不划算。x 版的正确用法都是**低频、关键时刻**的精修，不是替换。

## 建议的启动路径

等身份识别准确率出现具体瓶颈时，按 **3 → 1 → 2 → 4** 的顺序推进：先用 x 版在真实数据上量化 n 版的差距（用途 3 的评测部分），数据说话，再决定是否把 x 版接回入库链路。接回去的改动量不大：`config.py` 加一个类似 `yolo_accurate_model = "yolo11x-pose.pt"` 的配置项，`YoloPoseDetector` 本身是模型路径参数化的，直接复用。
