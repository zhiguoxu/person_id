我们的目标是给机器人提供人物识别系统，我们给机器人提供了每秒一帧的视觉图片，机器人会看到不同的人，他需要识别不同的人，并返回人物的ID。
提供一下方案建议：
1. 人脸识别的基本方案采用MediaPipe 把人脸识别裁剪出来，然后再用 InceptionResnetV1 embedding，后续就通过对比 embedding 识别是不是同一个人脸，当然我们还要考虑不同角度的人脸。
2. 人物追踪：充分利用视觉的时空连续性，比如当一个人脸部转动的时候，他的侧脸和正脸是有差异的，但是这个差异有很多中间步骤，这些中间步骤直接的差异比两端的差异小，这一点是可以利用的。
3. 人物特征特征分析： 比如衰变周期，比如人脸是长期特征，衣服可能每天换，有多套衣服多个并行特征，再比如特征权重，人脸是一个非常强的特征，达到一定阈值可以直接走捷径。
具体细节参考 implementation_plan.md，另外也借鉴一下行业内的方案，在 research_report.md。
4. 使用 python + web(react)，各种阈值可视化配置。
5. 请用前后端分离的web方案作为视频采集测试接口，并且提供可视化阈值控制，以及算法中间流程执行的可视化反馈。
6. 我们有足够的 CUDA 算力，永远是效果优先。
7. 人物追踪的时候要考虑多人，但是实际上只要返回最有可能和他对话的那个人。
