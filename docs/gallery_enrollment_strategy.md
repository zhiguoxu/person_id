# Gallery 入库策略设计

> 解释 face、body、wardrobe 三种特征的入库淘汰策略为何不同。

## 核心区别：同质竞争 vs 异质互补

| | face / body | wardrobe |
|:--|:--|:--|
| 每条记录是什么 | 同一外观的**不同角度快照** | **不同衣服**的特征 |
| 多条记录的关系 | 互相**竞争**（谁更清晰） | 互相**互补**（记住穿过哪些） |
| 例子 | 5 条都是"正面的张三" | "黑夹克、白T恤、蓝西装、灰卫衣、红外套" |

## 三种策略

### 1. Face — 时间衰减 + 质量替换（慢衰减）

```
衰减率: 0.005/天 (半衰期 ~100 天)
桶容量: MAX_FACES_PER_BUCKET = 5
```

- 同一姿态桶内的多条记录是**同质竞争**——都是"这个人从这个角度长什么样"
- 新帧的绝对质量 > 旧帧的衰减后质量 → 替换
- 衰减**慢**：人脸变化慢（发型、妆容、老化），高质量特征可以长期有效
- 如果不衰减：底库装满 `quality=0.99` 的完美特征后，发型变化将永远无法入库

### 2. Body — 时间衰减 + 质量替换（快衰减）

```
衰减率: 0.01/天 (半衰期 ~50 天)
桶容量: MAX_BODY_PER_BUCKET = 3
```

- 与 face 相同的竞争逻辑，但衰减**快 2 倍**
- 原因：人体外观受衣服影响大，换装后旧特征的价值快速下降
- 30 天前穿黑夹克时的高清正面特征（衰减后质量 0.69），应该被今天穿白T恤时的中等质量特征（0.85）替换

### 3. Wardrobe — EMA 精化 + 淘汰最久未见

```
匹配阈值: cos_sim > 0.85 (同一套衣服)
EMA 系数: α = 0.3
容量: MAX_OUTFITS = 20
```

- 多条记录是**异质互补**——每条代表一套不同的衣服
- **不能用质量竞争**：否则一件高清拍到的黑夹克会挤掉低清拍到的白T恤，下次穿白T恤就认不出了
- 同一套衣服（cos_sim > 0.85）：EMA 更新特征 + 重归一化，越穿越准
- 新衣服：追加；满了淘汰 `last_seen` 最早的（最久没穿 → 最不可能再穿）

## 为什么不能统一策略

### 如果 wardrobe 也用质量替换

```
底库: [黑夹克 q=0.95, 白T恤 q=0.72, 蓝西装 q=0.68]

今天穿白T恤出现, 检测质量 q=0.80:
  ✗ 质量替换: 白T恤 0.80 > 蓝西装 0.68 → 替换蓝西装
              下次穿蓝西装就认不出了!
  ✓ EMA 更新: 发现 cos_sim(白T恤, 0.72) = 0.91 > 0.85 → 更新白T恤特征
              蓝西装安全保留 ✓
```

### 如果 face/body 也用 EMA + 淘汰最久未见

```
底库 FRONTAL 桶: [清晰正面 q=0.95, 模糊正面 q=0.40, 遮挡正面 q=0.30]

今天拍到一帧清晰正面 q=0.92:
  ✗ EMA 更新: cos_sim(新帧, 模糊正面) = 0.88 > 0.85 → EMA 更新模糊正面
              模糊特征"被污染", 低质量帧永远占着位置!
  ✓ 质量替换: 0.92 > 0.30(遮挡) → 替换遮挡帧
              桶内质量稳步提升 ✓
```

## 总结

```
face/body: 同一个人的同一角度 → 谁更清晰谁留下 → 质量竞争
wardrobe:  同一个人的不同衣服 → 每件都是独立信息 → 多样性保留
```

策略差异的本质是**记录间的语义关系**不同，不是技术偏好。
