> **结论**：发现了若干关键架构风险和可补充技术，已据此修订最终方案。

## 一、关键发现：帧率对追踪的致命影响

### 1.1 问题背景

原始需求设定每秒一帧（1 FPS）。原方案选择 BoT-SORT 作为多目标追踪器。但经过深度调研发现，**所有主流 MOT 追踪器（DeepSORT、BoT-SORT、ByteTrack）都是在 25-30 FPS 视频上设计和评测的**，在低帧率下核心组件会严重退化甚至彻底失效。

### 1.2 定量分析

以室内场景为例（摄像头距离 3m、90° 水平视场角、1920px 宽画面），行走的人速度约 1.4 m/s：

| 帧率 | 帧间隔 (s) | 人移动距离 (m) | 像素位移 (px) | IoU (同一人) | 卡尔曼滤波可靠性 |
|---|---|---|---|---|---|
| 30 FPS | 0.033 | 0.047 | ~15 | 0.90+ | ✅ 极好 |
| 10 FPS | 0.1 | 0.14 | ~45 | 0.70+ | ✅ 好 |
| 5 FPS | 0.2 | 0.28 | ~90 | 0.45-0.55 | ⚠️ 勉强 |
| 3 FPS | 0.33 | 0.47 | ~150 | 0.15-0.30 | ❌ 不可靠 |
| **1 FPS** | **1.0** | **1.4** | **~450** | **0.0** | **❌ 完全失效** |

#### 三大失效机制

**1. IoU 归零**
- 人走了约 450 像素（画面宽度的 25%），前后两帧的检测框**完全不重叠**
- ByteTrack 纯靠 IoU 关联 → 直接报废
- BoT-SORT 第一阶段也是 IoU 匹配 → 同样失败

**2. 卡尔曼滤波预测无意义**
- 卡尔曼滤波的过程噪声协方差矩阵 Q 中，位置不确定性增长与 Δt⁴ 成正比
- 1 FPS vs 30 FPS：位置不确定性增长 **81 万倍** (30⁴ = 810,000)
- 预测不确定性（±250px）比人体检测框宽度（~200px）还大
- 预测等于随机猜测

**3. track_buffer 语义错误**
- `track_buffer=30` 在 30 FPS 下 = "丢失 1 秒后删除轨迹"
- 在 1 FPS 下 = "丢失 30 秒后删除轨迹"
- 语义完全错乱

### 1.3 结论

> **在 1 FPS 下，这不是一个"追踪 (Tracking)"问题，而是一个纯粹的"逐帧重识别 (Re-ID Matching)"问题。**

### 1.4 帧率方案对比

| 选项 | 追踪方案 | 效果预期 | 适用场景 |
|---|---|---|---|
| **A. 10+ FPS** ✅ 最终选择 | 双层架构 (轻量追踪 + 重模型识别) | 最优效果，Track ID 极其稳定 | 有高帧率摄像头 |
| B. 5 FPS | BoT-SORT (参数调整) | 较好，IoU 勉强可用 | 带宽受限 |
| C. 1 FPS | 自定义 ReID 关联器 | 可用但追踪稳定性差 | 硬件限制 |
| D. 双流 | 高帧率追踪 + 低帧率识别 | 最灵活 | 复杂架构 |

**最终决策：方案 A — 提高到 10+ FPS，使用双层处理架构。**

大多数 USB/IP 摄像头默认输出 30 FPS。如果当前的 1 FPS 是软件层面的降采样，只需修改采样率即可，是最简单且效果最好的路径。

## 三、补充技术分析

### 3.1 K-Reciprocal 重排序 → ✅ 采纳

**原理**：基于"互为近邻"的关系对初始 ReID 匹配结果重新排序。

**性能提升**（在 Market-1501 上）：

| 指标 | 重排序前 | 重排序后 | 提升 |
|---|---|---|---|
| Rank-1 (SOLIDER) | 96.9% | ~97.5% | +0.6% |
| mAP (SOLIDER) | 93.9% | ~96.2% | **+2.3%** |

**计算复杂度**：O(N²)，N = 底库人数
- N < 50 人（我们的场景）：**< 1ms**，完全可忽略
- 等于白赚 2-4% mAP 提升

**结论**：✅ 采纳。对小规模底库几乎零成本。

### 3.2 体型比例特征 → ✅ 采纳

利用 YOLO-Pose 已输出的 17 个关键点，**零额外模型开销**提取衣服无关的身体几何特征：
- 躯干/腿比例
- 肩宽/髋宽比例
- 手臂/躯干比例
- 头/身体比例

**特点**：
- ✅ 衣服无关（基于骨骼几何）
- ✅ 零额外计算（关键点已由 YOLO 输出）
- ⚠️ 区分力有限（同性别同体型的人难以区分）
- ⚠️ 对关键点精度敏感

**结论**：✅ 采纳作为辅助信号，权重较低。

### 3.3 声纹识别 (ECAPA-TDNN) → ⏳ 第二版

**调研结论**：

| 模型 | EER (VoxCeleb1) | 特征维度 | 参数量 | 推理速度 |
|---|---|---|---|---|
| ECAPA-TDNN | 0.87% | 192 | 6.2M | ~10ms/秒音频 |
| TitaNet-Large | 0.66% | - | 25M | 较快 |
| WavLM-Large | 0.38% | - | 300M | 较慢 |

**在机器人对话场景中的价值**：
- 声纹是与衣服无关、角度无关、光照无关的强身份特征
- EER 0.87% ≈ 99.1% 准确率，接近人脸识别
- 机器人对话场景天然产生音频信号
- 填补"背面、暗光、换衣"视觉盲区

**决策**：第二版再集成，先完成视觉通路。

### 3.4 质量加权时序特征聚合 → ✅ 已纳入方案

不使用单帧特征匹配底库，而是累积最近 N 帧的特征，用质量加权平均：

| 方法 | Rank-1 提升 | 复杂度 |
|---|---|---|
| 简单平均 | +1-2% | 极低 |
| 质量加权平均 | +3-4% | 低 |
| 注意力池化 | +4-5% | 中 |
| 视频 ReID (AP3D/PSTA) | +5-7% | 高 |

**结论**：采用质量加权平均，滑动窗口 5-10 帧。

### 3.5 主动感知建议系统 → ✅ 已纳入方案

机器人不是被动摄像头，可以主动行动改善识别效果：
- 背影 → 建议移动位置获取正脸
- 人脸质量低 → 建议靠近目标
- 身份不确定 + 对方说话 → 建议口头询问
- 完全陌生 → 建议打招呼

**实现方式**：回调/建议系统，不与视觉流水线紧耦合。

## 四、多模态融合策略分析

### 4.1 原方案问题

原方案使用人脸/全身固定 70/30 权重融合，过于僵硬。不同场景下各模态的可用性和可靠性差异巨大。

### 4.2 场景覆盖分析

```
各模态在不同场景下的可用性:

┌─────────────┬──────┬──────┬──────────┐
│   场景      │ 人脸 │ 全身 │ 体型比例 │
├─────────────┼──────┼──────┼──────────┤
│ 正面+近距离 │ ✅   │ ✅   │ ✅       │  → 三重验证, 最高置信度
│ 正面+远距离 │ ⚠️   │ ✅   │ ⚠️       │  → 人脸质量降低
│ 侧面       │ ⚠️   │ ✅   │ ✅       │  → 人脸可用但降权
│ 背面       │ ❌   │ ✅   │ ⚠️       │  → 仅靠全身, 最弱场景
│ 换衣+正面  │ ✅   │ ❌   │ ✅       │  → 人脸不受影响
│ 换衣+背面  │ ❌   │ ❌   │ ⚠️       │  → 几乎不可能识别
└─────────────┴──────┴──────┴──────────┘

第二版加入声纹后，"背面+对话"和"暗光"场景将得到覆盖。
```

### 4.3 最终融合策略：自适应权重

```python
# 各模态基础权重 (满质量时)
BASE_WEIGHTS = {
    'face':       0.50,   # 人脸: 最强身份信号
    'body':       0.35,   # 全身 ReID: 中等信号
    'proportion': 0.15,   # 体型比例: 辅助信号
}

# 捷径机制:
# - 人脸质量 > 0.7 且匹配度 > 0.75 → 直接确认，不需融合
# - 权重按模态可用性和质量动态归一化
```
