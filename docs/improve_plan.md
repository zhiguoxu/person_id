二、轨迹段聚合 — 按需触发 + 质量池
设计思路
FrameBuffer 维护每个 track 生命周期内质量最高的 N 帧（由 Tier 1 每帧 push + 质量评估）：
┌───────────────── FrameBuffer (质量池) ────────────────┐│                                                     ││  quality_pool: Top-N 帧 (N=10, 可配置)               ││    按质量排序, 新帧质量 > 最差帧时替换                ││    插入约束: 与池中已有帧最小间隔 ≥ 0.3s             ││      (若已有同时段帧, 仅当新帧更优时替换)            ││    堆维护: O(log N) / 帧                              ││                                                     ││  last_push_time: float                               ││    track 存活检测用 (不存储帧数据)                   ││                                                     ││  select_frames(): 直接返回整个 pool (零计算)         ││  get_best_frame(): pool[0] (被动探测用)              │└────────────────────────────────────────────────────┘
设计原则：池 = 选帧结果
- pool_size = Tier2 所需帧数 = 10，没有多余的"候补"
- select_frames() 直接返回整个 pool，无需二次筛选
- 插入时 ≥0.3s 间隔保证时间多样性（10 帧 × 0.3s → 覆盖 ≥3s）
TIP
零额外成本：质量评估本来每帧都要算（face_shortcut 需要），维护 quality_pool 只多了一次 O(log N) 堆比较，完全免费。
参数作用值quality_pool_size池容量 = Tier2 选帧数10pool_min_interval池内帧最小时间间隔 (保证多样性)0.3srefresh_intervals触发调度：下一次刷新何时发生(5, 10, 20)s
置信度体系
核心原则：只有正向匹配能达到"笃定"——你可以确信"就是这个人"（高分+高 margin），但永远无法确信"不是任何人"（也许他只是还没转过脸来）。因此 DEFINITE 是唯一的终态。
                    唯一终态                      ↓  DEFINITE ──── CONFIDENT ──── SUSPECTED ──── STRANGER   ≥A (停止)      ≥B (递减)     ≥C (重试)      <C (递减)                        ↓                          ↓                   达到 max_refresh_count 后                         ↓                          ↓                   ══════ 切换为被动监控 ═══════                   (轻量 ReID, 20s 周期, ~15ms)
级别名称条件主动阶段行为A笃定 (DEFINITE)top ≥ A (0.85) AND margin ≥ A_margin (0.20)✅唯一终态，永久停止B确定 (CONFIDENT)top ≥ B (0.72) AND margin ≥ B_margin (0.10) AND 唯一超过 B🔄 递减刷新C怀疑 (SUSPECTED)C (0.55) ≤ top < B⚠️ 最小间隔重试 (T₁=5s)—陌生人 (STRANGER)top < C (含空库)🔄 递减刷新
IMPORTANT
不对称设计：只有 DEFINITE 能停止一切。STRANGER 不是终态——即使当前分数极低，也可能是因为人背对摄像头。被动监控会捕捉变化并重新触发完整识别。
人脸快速通道 (Face Shortcut) → DEFINITE
高质量人脸匹配可在多帧聚合之前直接达到 DEFINITE，跳过整个 Tier2 流程。
pythondef check_face_shortcut(face_score: float, face_quality: float,                        margin: float, config: MatchingConfig) -> bool:    """人脸快速通道: 在 Tier1 中检查, 多帧聚合之前        触发条件 (全部满足):      1. 人脸质量极高 (≥ 0.85)      2. 人脸匹配分极高 (≥ A_threshold = 0.85)      3. 与第二名 margin 足够大 (≥ A_margin = 0.20)        满足时直接设为 DEFINITE, 不经过 Tier2 流程.    """    return (face_quality >= config.face_shortcut_quality and    # 0.85            face_score >= config.A_threshold and                # 0.85            margin >= config.A_margin)                          # 0.20
Tier1 每帧处理流程:  检测 → 追踪 → 人脸质量评估                    ↓          face_shortcut 检查            │ 满足          │ 不满足            ↓               ↓         DEFINITE ✅     正常 Tier2 流程         (永久停止)      (主动/被动两阶段)
TIP
为什么 face_shortcut 独立于 Tier2? 高质量正面人脸的匹配准确率 > 99%，多帧聚合只会增加延迟而不会显著提高准确率。让机器人在 1 帧内就认出熟人，提供最快的交互体验。
两阶段监控：避免卡死
所有非 DEFINITE 的 track 都经历两个阶段，保证没有任何状态会永久卡死：
┌─────────────────── 主动阶段 (Active) ──────────────────┐│  完整 Tier2 (多帧聚合 + 全模态)                          ││  间隔: 5s → 10s → 20s (最多 max_refresh_count 次)       ││  成本: ~110ms / 次                                       ││  目标: 尽快达到 DEFINITE                                  ││                                                          ││  SUSPECTED: 使用 T₁ 最小间隔 (不逐帧重试, 避免风暴)     │└────────────────────────┬─────────────────────────────────┘                         │ 达到 max_refresh_count                         ▼┌─────────────────── 被动阶段 (Passive) ─────────────────┐│  轻量单帧 ReID (buffer 最优帧 + 快速匹配)                 ││  间隔: passive_interval (20s)                            ││  成本: ~15ms / 次                                        ││                                                          ││  分场景触发升级:                                          ││    STRANGER:  探测分 ≥ 怀疑线(0.40) / 新检测到人脸       ││    CONFIDENT: 分数下降≥0.20 / 匹配对象变 / 分数上升≥0.10 ││  → 重置 tier2_count, 重回主动阶段                        │└──────────────────────────────────────────────────────────┘
被动探测设计
python@dataclassclass PassiveProbe:    """被动监控状态"""    last_top_score: float = 0.0    last_best_match_id: str | None = None    last_probe_time: float = 0.0    last_frame_id: int = -1              # 避免重复使用同一帧    had_face: bool = False# ─────────────────────────────────────────────# 动态修正: Probe → Tier2 分数偏移校准# ─────────────────────────────────────────────class ProbeCalibrator:    """跟踪 probe 与 Tier2 之间的分数偏移, 自适应修正        原理:      被动探测(单帧) 比完整 Tier2(多帧聚合) 系统性地低 ~0.05-0.15。      这个偏移量受模型、光照、分辨率等因素影响，不应硬编码。          方法:      每次完整 Tier2 完成后, 如果之前有 probe 记录, 计算实际偏移:        delta = tier2_score - last_probe_score      用 EMA 跟踪这个 delta, 作为 probe 分数的动态修正量。        使用:      probe 判断时: corrected_score = probe_score + calibrator.offset      这个 corrected_score 用于与阈值比较, 而非原始 probe_score。    """        def __init__(self, initial_offset: float = 0.10, alpha: float = 0.3,                 min_offset: float = 0.0, max_offset: float = 0.25):        self._offset = initial_offset    # 初始预估 (经验中位数)        self._alpha = alpha              # EMA 平滑系数 (越大越跟踪最新)        self._min_offset = min_offset    # 下界 (probe 不可能比 Tier2 好)        self._max_offset = max_offset    # 上界 (偏移过大说明异常)        self._n_samples = 0              # 已观测样本数        @property    def offset(self) -> float:        return self._offset        def observe(self, probe_score: float, tier2_score: float):        """Tier2 完成后调用, 记录实际偏移                只在以下条件同时满足时调用:        1. 该 track 在 Tier2 之前有 probe 记录        2. probe 和 Tier2 匹配到同一个人 (best_match_id 一致)        3. 两次采样时间间隔 < 30s (环境未显著变化)        """        delta = tier2_score - probe_score        delta = max(self._min_offset, min(self._max_offset, delta))  # clip                if self._n_samples == 0:            self._offset = delta    # 第一次直接赋值        else:            self._offset = self._alpha * delta + (1 - self._alpha) * self._offset        self._n_samples += 1# per-camera 实例 — 偏移取决于摄像头的物理条件 (分辨率/光照/距离)# 不同摄像头的 probe-tier2 delta 差异显著:#   室内近距 (delta~0.07) vs 门口逆光 (delta~0.15)# 全局单例会平均化差异, 对每个摄像头都不准确# 每个 CameraPipeline 持有自己的 ProbeCalibrator 实例

置信度判定逻辑
pythondef resolve_reid(self, match_result: MatchResult) -> MatchResult:    """四级置信度判定 (无笃定陌生)"""    top = match_result.top_score    margin = match_result.margin    cfg = self._config        # A 级: 笃定 — 唯一终态    if top >= cfg.A_threshold and margin >= cfg.A_margin:        match_result.status = IdentityStatus.DEFINITE        match_result.best_match = match_result.candidates[0]        return match_result        # B 级: 确定 — 要求高分 + 高区分度 + 无竞争者    if top >= cfg.B_threshold:        n_above_B = sum(1 for c in match_result.candidates                        if c.fused_score >= cfg.B_threshold)        if n_above_B > 1:            # 多人超过 B 线 → 冲突            match_result.status = IdentityStatus.CONFLICT            match_result.best_match = match_result.candidates[0]        elif margin >= cfg.B_margin:            # 唯一超过 B + margin 足够 → 确定            match_result.status = IdentityStatus.CONFIDENT            match_result.best_match = match_result.candidates[0]        else:            # 唯一超过 B 但 margin 不足 → 降级为怀疑            match_result.status = IdentityStatus.SUSPECTED            match_result.best_match = match_result.candidates[0]        return match_result        # C 级: 怀疑    if top >= cfg.C_threshold:        match_result.status = IdentityStatus.SUSPECTED        match_result.best_match = match_result.candidates[0]        return match_result        # <C: 陌生人 (包括空库、分数极低等所有情况)    match_result.status = IdentityStatus.STRANGER    return match_result

刷新间隔推导
之前的 5s/10s/30s 设定缺乏推导依据。下面从三个物理约束出发推导每个间隔。
约束 1：对话响应时间 → 第 1 次刷新
第 0 次识别: track 刚出现, buffer 可能只有 1~3 帧第 1 次刷新: 应在对话启动窗口内完成, 确保机器人能及时认出人→ T₁ = 5 秒 (对话启动延迟)
推导：人走向机器人 → 开始说话 → 机器人在 5 秒内完成首次完整确认。此时 quality_pool 已积累 5s × 12fps 中最优的若干帧。第 1 次刷新是对初始判断的首次完整确认，信息增益最大。
约束 2：独立观测窗口 → 第 2 次刷新
第 1 次刷新使用 [0s, 5s] 的数据第 2 次刷新应使用与第 1 次完全不重叠的新窗口 [5s, 10s]→ T₂ = 10 秒 (两次独立观测)
推导：两次独立观测得出一致结果，误判概率从 P 降至 P²。此时 quality_pool 已积累 10s 内最优帧，可用数据更充分。如果两次都判为 CONFIDENT（P_err ≈ 5%），误判降到 0.25%。
约束 3：场景变化时间尺度 → 稳态刷新
稳态刷新目的: 捕捉 tracker ID switch在机器人对话场景 (1-5 人, 少遮挡):  ID Switch 较稀少, 典型间隔 > 60 秒  主要发生在: 人穿越遮挡、走出画面再回来→ T_steady = 20s (与被动探测间隔对齐, 主动→被动过渡平滑)
推导：类比 Nyquist 采样定理，ID switch 平均间隔 ~60s，刷新 ≤ 30s 即可。取 T_steady = 20s 兼顾响应性，同时与被动探测 (20s) 对齐，使主动阶段最后一级和被动阶段无缝衔接。
注意力目标差异化
pythondef get_refresh_interval(track: TrackedPerson, tier2_count: int,                          config: MultiFrameConfig) -> float:    """刷新间隔与 buffer 解耦, 使用独立配置的 refresh_intervals"""    intervals = config.refresh_intervals                   # (5, 10, 20)        if tier2_count <= 1:        interval = intervals[0]           # 5s  (对话响应)    elif tier2_count == 2:        interval = intervals[1]           # 10s (观测独立性)    else:        interval = intervals[2]           # 20s (稳态监控)        # 注意力目标加速    if track.is_current_target:        interval *= config.attention_factor               # 0.6        return interval

三、核心问题：Gallery 相似度计算策略
问题分析
提取出查询特征 Q(face, body) 后，需要计算它与 Gallery 中每个人 P 的相似度。每个人 P 可能有 多个角度的多张照片。
Gallery 人物 P:  face_features:    FRONTAL: [f₁ (q=0.92), f₂ (q=0.85), f₃ (q=0.78)]    LEFT:    [f₄ (q=0.70)]    RIGHT:   [f₅ (q=0.65), f₆ (q=0.55)]    BACK:    []  wardrobe:    outfit₁(黑衣): [b₁]    outfit₂(白衣): [b₂]  body_features (新增, 按姿态分桶):    FRONTAL: [b₃ (q=0.88), b₄ (q=0.72)]    LEFT:    [b₅ (q=0.65)]    ...查询 Q (姿态=LEFT):  face_embedding: q_face (512-dim)  body_embedding: q_body (2048-dim)
三种方案对比
方案 A：纯 Max-Pooling（当前实现）
pythonscore(Q, P) = max{ cos_sim(Q, gallery_entry_i) × decay_i }  for all i
✅ 优点  简单高效，O(N) 复杂度  不丢失信息：每条 gallery entry 独立保留  不受"幻影中心"问题影响  思路: 在所有 gallery 条目中找到与 query 最相似的那一条，用其相似度作为该人的匹配分。
方案 B：Gallery 特征融合（质心匹配）
pythoncentroid_P = weighted_average(all gallery entries of P)score(Q, P) = cos_sim(Q, centroid_P)
✅ 优点  抗噪声：多条 entry 平均后更鲁棒  快速：只需与一个质心做比较  适合人脸（外观稳定的模态）  思路: 将一个人的所有 gallery 特征预先融合为一个代表性质心，查询时只需比较一次。
方案 C：Viewpoint-aware / Pose-guided（全量视角感知）
python# Pose-guided Feature Alignmentaligned_gallery = align_to_canonical_view(gallery_entries)# 或 Viewpoint-aware attentionscore = attention_weighted_sum(Q, gallery_entries, pose_Q, pose_entries)
✅ 优点  理论最优：完全考虑视角差异  Pose-guided alignment 可将不同视角特征对齐到统一空间  Attention 机制可学习最优加权策略  思路: 使用额外的姿态/视角信息来动态对齐或加权 gallery 特征。

最终推荐：分桶质心 + 跨桶 cos_sim Max-Pooling（无亲和力加权）
层级策略解决的问题桶内（同视角多样本）质量加权质心利用多样本的统计优势，抗噪声跨桶（不同视角）纯 cos_sim Max-Pooling让模型自身判断跨姿态相似度人脸 vs 人体分别处理人脸稳定可做质心；人体换装需保留独立 outfit
IMPORTANT
为什么不需要姿态亲和力矩阵 (POSE_AFFINITY)?
ReID 模型的 cos_sim 已经隐式编码了跨姿态的外观退化：
- 同姿态匹配：cos_sim ≈ 0.90（信号强）
- 邻近姿态匹配：cos_sim ≈ 0.78（自然退化）
- 不同人：cos_sim ≈ 0.30（无论什么姿态）
外加 affinity 是双重惩罚，会把正确的跨姿态匹配压到阈值以下，降低召回。 多帧全桶交叉 max-pooling 已经保证了最优姿态组合自然胜出。
人脸匹配：桶内质心 + 跨桶 cos_sim Max-Pooling
人脸外观在短期内是稳定的（不会换脸），因此同视角下的多张照片可以安全地做质心融合。
pythondef match_face_v2(self, query_faces: dict[PoseBucket, tuple[np.ndarray, float]],                  gallery: dict[str, PersonProfile]) -> list[MatchCandidate]:    """    人脸匹配 — 查询端全桶×Gallery端全桶交叉匹配:    1. 对每个人的每个 Gallery 姿态桶，计算桶内质量加权质心    2. 查询端每个桶 × Gallery 每个桶: 纯 cos_sim    3. 取所有 (query_bucket, gallery_bucket) 组合中的最大分    """    now = time.time()    half_life = self._gallery_cfg.face_half_life_days    candidates = []        for person_id, profile in gallery.items():        best_weighted_score = -1.0                for bucket, entries in profile.face_features.items():            if not entries:                continue                        # ======= 桶内质心 =======            # 质量 × 时间衰减 作为融合权重            weights = []            embeddings = []            for entry in entries:                decay = entry.time_decay_weight(now, half_life)                w = entry.quality_score * decay                weights.append(w)                embeddings.append(entry.embedding)                        weights = np.array(weights)            if weights.sum() < 1e-8:                continue                        # 质量加权质心            embeddings_mat = np.stack(embeddings)        # (N, 512)            centroid = np.average(embeddings_mat, axis=0, weights=weights)            centroid = centroid / (np.linalg.norm(centroid) + 1e-8)                        # ======= cos_sim Max-Pooling (查询全桶 × Gallery 单桶) =======            for q_pose, (q_emb, q_quality) in query_faces.items():                cos_sim = float(np.dot(q_emb, centroid))                best_weighted_score = max(best_weighted_score, cos_sim)                if best_weighted_score > 0.0:            candidates.append(MatchCandidate(                person_id=person_id,                display_name=profile.display_name,                face_score=best_weighted_score,            ))        candidates.sort(key=lambda c: c.face_score or 0.0, reverse=True)    return candidates
IMPORTANT
为什么人脸可以做桶内质心？ 同一姿态下的人脸特征分布近似单峰高斯分布（同一个人的正面照之间差异很小），质心恰好落在分布中心，是最优代表点。而跨姿态的特征分布是多峰的（正面和背面完全不同），质心会落在"峰间空洞"，所以跨桶不能做质心，必须用 max-pooling。
人体匹配：双层匹配（姿态分桶 + 服装库）
人体与人脸不同：换装会导致同一姿态桶内的特征出现多峰分布（黑衣和白衣的正面特征完全不同）。因此人体不能做桶内质心。
pythondef match_body_v2(self, query_bodies: dict[PoseBucket, tuple[np.ndarray, float]],                  gallery: dict[str, PersonProfile]) -> list[MatchCandidate]:    """    人体匹配 — 查询全桶×Gallery双层交叉匹配:        Layer 1: 姿态分桶匹配 (body_features)      - 查询每个桶 × Gallery 每个桶: 纯 cos_sim × decay      - 取所有组合的 max        Layer 2: 服装库匹配 (wardrobe)      - 查询每个桶 × 所有 outfit: 独立比较      - 取 max × 近因权重        最终分: max(layer1_score, layer2_score)    """    now = time.time()    candidates = []        for person_id, profile in gallery.items():        scores = []                # ======= Layer 1: 查询全桶 × Gallery 全桶 =======        for bucket, entries in profile.body_features.items():            for entry in entries:                decay = entry.time_decay_weight(now, half_life_days=30.0)                for q_pose, (q_emb, q_quality) in query_bodies.items():                    cos_sim = float(np.dot(q_emb, entry.embedding))                    score = cos_sim * decay                    scores.append(score)                # ======= Layer 2: 服装库匹配 =======        for outfit in profile.wardrobe:            for q_pose, (q_emb, q_quality) in query_bodies.items():                cos_sim = float(np.dot(q_emb, outfit.body_embedding))                recency = outfit.recency_weight(now)                score = cos_sim * recency                scores.append(score)                if scores:            best_score = max(scores)            if best_score > 0.0:                candidates.append(MatchCandidate(                    person_id=person_id,                    display_name=profile.display_name,                    body_score=best_score,                ))        candidates.sort(key=lambda c: c.body_score or 0.0, reverse=True)    return candidates

BodyProportions 数据结构
python@dataclassclass BodyProportions:    torso_leg_ratio: float      # 躯干/腿长  (典型值 0.75-0.95)    shoulder_hip_ratio: float   # 肩宽/髋宽  (典型值 1.2-1.6)    arm_torso_ratio: float      # 手臂/躯干  (典型值 0.9-1.2)    head_body_ratio: float      # 头/身体    (典型值 0.12-0.18)    relative_height_px: float   # 帧内像素高度 (仅用于质量过滤, 不参与匹配)        def to_vector(self) -> np.ndarray:        """转为 4-dim 向量 (不含 relative_height_px)"""        return np.array([            self.torso_leg_ratio,            self.shoulder_hip_ratio,            self.arm_torso_ratio,            self.head_body_ratio,        ], dtype=np.float32)
NOTE
relative_height_px 是帧内绝对像素高度，受距离影响剧烈，不参与相似度计算。它只用于质量过滤（太小的人关键点不可靠）。
高斯核相似度
python@staticmethoddef similarity(a: BodyProportions, b: BodyProportions) -> float:    """    高斯核相似度: exp(-||a - b||² / (2σ²))        输入: 两个 4-dim 向量 (torso_leg, shoulder_hip, arm_torso, head_body)    输出: [0, 1], 1 = 完全相同        σ = 0.15, 物理含义: 允许约 15% 的比例差异仍视为相似    """    va = a.to_vector()        # (4,)    vb = b.to_vector()        # (4,)    diff = va - vb            # (4,) 逐维差异    sigma = 0.15    sq_dist = np.dot(diff, diff)  # ||a-b||² = Σ(aᵢ-bᵢ)²    return float(np.exp(-sq_dist / (2 * sigma * sigma)))
特性分析
score = exp(-||diff||² / 0.045)示例:  4维均差 0.00 → score = 1.000  (完全匹配)  4维均差 0.05 → ||diff||² = 4×0.05² = 0.01  → score = 0.800  4维均差 0.10 → ||diff||² = 4×0.10² = 0.04  → score = 0.411  4维均差 0.15 → ||diff||² = 4×0.15² = 0.09  → score = 0.135  (σ边界)  4维均差 0.20 → ||diff||² = 4×0.20² = 0.16  → score = 0.029  (几乎为0)
特性说明无需训练纯几何计算，不依赖任何模型衣服无关骨骼比例不因换装改变光照无关不依赖像素外观低区分度体型差异小的人分数接近，所以基础权重仅 15%无姿态依赖同一个人任何角度的骨骼比例理论上一致无时间衰减生理特征长期稳定，Gallery 存储累积平均
match_proportions 完整伪代码
pythondef match_proportions(self, query_proportions: BodyProportions,                      gallery: dict[str, PersonProfile]) -> list[MatchCandidate]:    """体型比例匹配 — 高斯核, 无姿态/时间维度"""    candidates = []    for person_id, profile in gallery.items():        if profile.body_proportions is None:            continue        score = BodyProportions.similarity(query_proportions, profile.body_proportions)        candidates.append(MatchCandidate(            person_id=person_id,            display_name=profile.display_name,            proportion_score=score,        ))    candidates.sort(key=lambda c: c.proportion_score or 0.0, reverse=True)    return candidates
